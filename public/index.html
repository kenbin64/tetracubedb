◈ Dimensional Architecture
THE 7 DIMENSIONS

Every piece of data — scalar, array, table, or event stream — lives at a specific dimensional level on the gyroid surface. Dimensions are not abstractions; they are geometric coordinates.

0
VOID
null / empty set
The empty set. A cell that exists as a placeholder without value. The geometric origin. Represents absence, not deletion.
∅ → no value, no coordinates
1
POINT
scalar · number · string · bool
A single entity. A discrete value at one coordinate on the gyroid surface. The atom of all data.
score = 9999 → gx=5.32, gy=3.88, gz=20.69
2
LINE
array · column
An ordered sequence of POINT values forming a column. One-dimensional data structured along the gyroid's z-axis gradient.
[1, 2, 3, 5, 8, 13] → Fibonacci column
3
WIDTH
object · row · record
A row of named fields. The WIDTH dimension spans multiple coordinates at the same row key across the surface manifold.
{username, score, avatar} → player record
4
PLANE
x × y · one 2D slice · domain alias: table / frame / face
The universal 2D slice. What you call it depends on your domain: data uses a table, animation uses a frame, 3D geometry uses a face or layer, a 3D mesh builds from vertices (POINT) → edges (LINE) → faces (PLANE). All are the same dimension. All map as z = x·y on the gyroid.
data: players table · animation: keyframe · geometry: mesh face
5
STACK
z-axis · stack of delta-planes · domain alias: sequence / timeline / layers
Each step up z is another PLANE recording only what changed. Nothing unchanged is repeated — it is implied by the plane below. Top = current state. Below = history of how it got there. You append, never mutate. In data: delta-rows. In animation: frames. In geometry: layers or vertex sequences. Same dimension.
data: delta stack · animation: frame sequence · geometry: mesh layers · fib=5
6
VOLUME
complete body of the stack · total measured extent
The full body formed by all delta-planes stacked along z — the volume. Not the object itself; the measured aggregate of everything inside it. VOLUME is a single point of M. fib=8.
entire namespace · full session history · all deltas combined · fib=8
7
M
the whole object · identity · discrete
A complete, discrete, addressable system — treated as a single POINT by anything above it. Identity, not contents. Atomic from the outside. To access contents: change scope (O(1)). fib=13.
a database instance · a microservice · a game · fib=13

THE GEOMETRY

The primitive is z = x·y. Rotate that 90° and you get a helix. A lattice of helices is the Schwartz Diamond Gyroid:

z = x·y  →  90°  →  helix  →  lattice  →  F(x,y,z) = 0

The full lattice surface is the database. Every cell lives on the surface. 8 Fibonacci dimensions (0,1,1,2,3,5,8,13), each Dₙ a single point of Dₙ₊₁. D5 = STACK is the z-axis: a stack of delta-planes, each recording only what changed. D6 = VOLUME is the complete measured body of that stack. D7 = M is the whole object — identity, not contents.

The dimension is universal — the label is a domain alias. What data calls a table, animation calls a frame, and 3D geometry calls a face are all D4 PLANE. A mesh builds exactly as the model predicts: vertex (POINT) → edge (LINE) → face (PLANE) → layers (STACK) → full mesh (VOLUME) → object (M). Every domain maps onto the same 8 dimensions and the same gyroid surface. z = x·y is the primitive for all of them.

The delta model: you never mutate a record. You add a new PLANE (a frame) with only the fields that changed. The top plane is always the current state. Every plane below shows how it got there. Replaying the stack forward reconstructs any moment. Querying the top gives O(1) current state without scanning history.

Every M is atomic and discrete from the outside. To access its contents you do not traverse it — you change scope. Knock + handshake → you enter a new coordinate space at the next scale down. The outer M is forgotten. O(1) at every level. The namespace is the scope. The same 8 dimensions repeat at every scale, from a single cell to the entire internet. The pattern does not break at any scale.

Each Dₙ is a single point of Dₙ₊₁. D5=STACK is the z-axis of delta-planes. D6=VOLUME is the complete measured body of the stack. D7=M is the whole object (identity). 13/8=1.625→φ. One M = one POINT at the next scale.
DELTA STACK · APPEND-ONLY · TOP = CURRENT STATE
z₊₁ = PLANE(only changed fields) · z_top = current · z₀…ₙ₋₁ = history
Never mutate. Add a new PLANE with only what changed. Unchanged fields are implied by the plane below — not repeated. Top of the stack = current state. Replay any slice for point-in-time queries. Not rows updated: planes appended. This is dimensional programming.
KNOCK + HANDSHAKE · topology public, content private
knock = SHA-256( myAddr + ":" + neighborAddr + ":" + nonce )
Every node knows neighbors' surface addresses — computable from F=0. Content requires: knock (present address) → handshake (owner grants scoped token). Secure by geometry, not by obscurity.

◈ Architecture
HOW IT WORKS
01
📐
GYROID COORDINATES
Your address (namespace/table/row/col) is hashed into (gx,gy) using djb2. gz₀ = gx·gy seeds the Relation Surface. Newton-Raphson solves F=0 in 4 iterations.
02
🗄️
PERSISTENT CELLS
Values are stored in SQLite at /var/lib/tetracubedb — outside the app directory. Zero data loss on redeploy. Gyroid coordinates stored alongside every cell.
03
⚡
MANIFOLD FUNCTIONS
Register handlers at patterns like "ns:table:col" with wildcards. Every write invokes matching functions — auth, lobby state machines, leaderboard ranking, session replay.
04
📡
REAL-TIME EVENTS
WebSocket subscriptions fire whenever a matching cell changes. Clients subscribe per table or namespace. No polling. Push-only live updates.
05
🔐
BEARER AUTH
API clients authenticate with client_id:api_key Bearer tokens. bcrypt-hashed keys. Namespace-scoped access control. Admin flag for system operations.
06
🌐
MULTI-TENANT
Multiple applications share one TetracubeDB instance via namespaces. kensgames.com is a tenant. Your application can be next. Complete namespace isolation.

REST + WebSocket API
# Store a POINT (D1) — value gets gyroid coordinates automatically
POST /v1/cell/:namespace/:table/:row/:col
Authorization: Bearer client_id:api_key
Content-Type: application/json

{ "value": 9999 }

# Response — includes gyroid surface coordinates
{
  "ok": true,
  "addr": "497574a07dc87faf...",   // SHA-256 surface address
  "gx": 5.325810, "gy": 3.885794, "gz": 20.695005,
  "dim": 1,           // POINT
  "processed": [{ "pattern": "auto-delta-tracker", "ok": true }]
}
# Get a single cell
GET /v1/cell/kensgames/players/user-ken/score
Authorization: Bearer client_id:api_key

→ { "value": 9999, "dim": 1, "dim_name": "point", "gx": ..., "gy": ..., "gz": ... }

# Get entire row — returns D3 WIDTH record
GET /v1/row/kensgames/players/user-ken

→ { "_row": "user-ken", "_dim": 3, "score": 9999, "username": "KenMaster",
    "_gyroid": { "gx": 5.32, "gy": 3.88, "gz": 20.69 } }

# Scan full table — returns D4 PLANE
GET /v1/table/kensgames/players

# Gyroid radius query — find all cells within surface distance
POST /v1/query/radius
{ "namespace": "kensgames", "gx": 5.3, "gy": 3.9, "gz": 20.7, "radius": 2.0 }
# Push delta to D5 STACK — temporal / event sequence
POST /v1/stack/kensgames/score_history/user-ken/score
{ "delta": { "score": 12500, "ts": 1745001000 } }

→ { "ok": true, "seq": 2, "gz": 23.387 }   // gz encodes the Relation Surface z=x·y

# Read stack — full temporal series
GET /v1/stack/kensgames/score_history/user-ken/score

→ {
    "_dim": 5,                   // STACK
    "frames": [
      { "seq": 1, "delta": { "score": 9999 },  "at": 1776787769 },
      { "seq": 2, "delta": { "score": 12500 }, "at": 1776787770 }
    ]
  }
// Connect WebSocket
const ws = new WebSocket('wss://tetracubedb.com/ws');

ws.onopen = () => {
  // Authenticate
  ws.send(JSON.stringify({
    type: 'auth',
    client_id: 'your-client-id',
    api_key:   'your-api-key'
  }));
};

ws.onmessage = e => {
  const msg = JSON.parse(e.data);

  if (msg.type === 'auth_ok') {
    // Subscribe to a table
    ws.send(JSON.stringify({ type: 'subscribe', table: 'kensgames:players' }));
  }

  if (msg.type === 'set') {
    console.log('live update', msg.row, msg.col, msg.value);
  }
};
// Node.js or browser UMD client
const { TetracubeClient } = require('./tetracube_client');

const db = new TetracubeClient({
  url:      'https://tetracubedb.com',
  clientId: 'your-client-id',
  apiKey:   'your-api-key',
  namespace:'myapp'
});

// Write a POINT (D1)
await db.set('users', 'alice', 'score', 4200);

// Read a row (D3 WIDTH)
const player = await db.getRow('users', 'alice');

// Push to stack (D5 STACK)
await db.pushDelta('events', 'alice', 'action', { move: 'jump', t: Date.now() });

// Real-time subscription
db.subscribe('users', ({ row, col, value }) => console.log(row, col, value));
await db.connectWS();
