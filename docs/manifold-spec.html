THE RELATION SURFACE

§4 — SUBSTRATE ARCHITECTURE: THE DIAMOND PATTERN

4.1 What Is a Substrate?

A substrate is a self-contained computational module that:

  • Owns a single domain (audio, board state, rules, AI, etc.)
  • Evaluates coordinates on shared manifold surfaces
  • Publishes its outputs as coordinates on those surfaces
  • Reads inputs by sampling coordinates published by other substrates
  • NEVER directly calls another substrate's functions
  • NEVER passes callbacks, events, or messages to other substrates
  • Has no knowledge of which other substrates exist

A substrate is not a "service," a "component," or a "module" in any traditional software sense. It is a geometric entity that occupies a region of the manifold and communicates exclusively through the surface topology. Its isolation is absolute — if every other substrate were removed from the system, a substrate would still run without error.

4.2 The Diamond Communication Pattern

Substrate A Manifold Surface Substrate B │ │ │ ├── publishes ─────────→│←──────── reads ──────┤ │ │ │ ├── reads ←────────────│──────────── publishes─┤ │ │ │ │ (the surface IS the medium) │ │ (no direct connection between A ↔ B) │

Substrates write coordinates. Other substrates read coordinates. The surface topology determines what information flows where. There is no dependency injection, no service locator, no observer pattern. There is only the surface.

4.3 Substrate Lifecycle

Every substrate follows the same lifecycle, without exception, on every frame:

  1. INITIALIZE — Set up internal state, register surface coordinates of interest. Happens once at startup.
  2. SAMPLE — Read coordinates from shared surfaces (input phase).
  3. EVALUATE — Process inputs through domain-specific logic.
  4. PUBLISH — Write output coordinates to shared surfaces.
  5. RENDER — If the substrate has visual/audio output, produce it.
PHASE ORDERING IS INVIOLABLE

ALL substrates SAMPLE before ANY substrate EVALUATES. ALL substrates EVALUATE before ANY substrate PUBLISHES. This guarantees frame consistency.

4.4 Surface Registry

The Surface Registry is the only shared infrastructure in the entire Manifold system. It is a flat dictionary mapping coordinate names to values. It has NO logic — it is pure data.

REGISTRY INVARIANTS

Single-Writer: Only one substrate MAY publish to a given coordinate. Violation SHALL throw an error at registration time.

Multi-Reader: Any number of substrates MAY read any coordinate.

No Dependency Graph: The registry does not know or care about read/write ordering.

Frame Consistency: All reads within a single frame see the same values (double-buffered). A substrate's writes in frame N are not visible to any reader until frame N+1.

5.3 Substrate Class Template

Every substrate MUST follow this exact class structure. The four lifecycle methods (sample, evaluate, publish, render) are the only interface contract.

// [name].substrate.js
import { Registry } from '../manifold/registry.js';
import { evaluateSurface } from '../manifold/surface.js';

export class NameSubstrate {
  constructor(registry) {
    this.registry = registry;
    // Declare published coordinates — this substrate is the SOLE writer
    this.registry.registerPublisher('domain.coordinate.name', this);
    // Internal state only — no references to other substrates
    this._inputValue = null;
    this._outputValue = null;
  }

  sample() {
    // Read coordinates from the registry — NEVER call another substrate
    const value = this.registry.read('other.coordinate.name');
    // Store locally for the evaluate phase
    this._inputValue = value;
  }

  evaluate() {
    // Domain-specific computation using manifold surface evaluation
    const z = evaluateSurface(this._inputValue, someOtherParam);
    this._outputValue = z;
  }

  publish() {
    // Write outputs to the registry
    this.registry.write('domain.coordinate.name', this._outputValue);
  }

  render(ctx) {
    // Optional: produce visual/audio output
    // Only substrates with user-facing output implement this
  }
}

// main.js — The heartbeat
import { Registry } from './manifold/registry.js';
import { BoardSubstrate } from './substrates/board.substrate.js';
import { RulesSubstrate } from './substrates/rules.substrate.js';
import { PegSubstrate } from './substrates/peg.substrate.js';

import { ThemesSubstrate } from './substrates/themes.substrate.js';
import { InputSubstrate } from './substrates/input.substrate.js';

const registry = new Registry();
const substrates = [
  new BoardSubstrate(registry),
  new RulesSubstrate(registry),
  new PegSubstrate(registry),
  new CardDeckSubstrate(registry),
  new MusicSubstrate(registry),
  new AudioSubstrate(registry),
  new AISubstrate(registry),
  new ThemesSubstrate(registry),
  new InputSubstrate(registry),
];

function tick(timestamp) {
  // Phase 1: SAMPLE — all substrates read from registry
  for (const s of substrates) s.sample();
  // Phase 2: EVALUATE — all substrates compute
  for (const s of substrates) s.evaluate();
  // Phase 3: PUBLISH — all substrates write to registry
  for (const s of substrates) s.publish();
  // Phase 4: Registry double-buffer swap
  registry.swap();
  // Phase 5: RENDER — visual/audio substrates produce output
  for (const s of substrates) s.render(ctx);
  requestAnimationFrame(tick);
}

5.5 Surface Registry Implementation

// registry.js
export class Registry {
  constructor() {
    this._front = new Map();       // Current frame (read buffer)
    this._back  = new Map();       // Next frame (write buffer)
    this._publishers = new Map();  // coordinate → substrate (single-writer)
  }

  registerPublisher(coordinate, substrate) {
    if (this._publishers.has(coordinate)) {
      throw new Error(
        `Coordinate "${coordinate}" already has a publisher. Single-writer violated.`
      );
    }
    this._publishers.set(coordinate, substrate);
    this._front.set(coordinate, null); // Initialize as Null state
    this._back.set(coordinate, null);
  }

  read(coordinate) {
    return this._front.get(coordinate); // Always read from front buffer
  }

  write(coordinate, value) {
    this._back.set(coordinate, value);  // Always write to back buffer
  }

  swap() {
    [this._front, this._back] = [this._back, this._front];
    // Copy front to back for next frame's baseline
    for (const [key, val] of this._front) {
      this._back.set(key, val);
    }
  }
}

5.6 Surface Evaluation Implementation

// surface.js
export function evaluateRelation(x, y) {
  return x * y;           // z = xy — the Relation surface
}

export function evaluateForm(x, y) {
  return x * y * y;       // z = xy² — the Form surface
}

export function gradientRelation(x, y) {
  return { dx: y, dy: x }; // ∇f = (y, x)
}

export function gradientForm(x, y) {
  return { dx: y * y, dy: 2 * x * y }; // ∇f = (y², 2xy)
}

export function evaluateSurface(x, y, surfaceType = 'relation') {
  switch (surfaceType) {
    case 'relation': return evaluateRelation(x, y);
    case 'form':     return evaluateForm(x, y);
    default:         return evaluateRelation(x, y);
  }
}

export function gradientSurface(x, y, surfaceType = 'relation') {
  switch (surfaceType) {
    case 'relation': return gradientRelation(x, y);
    case 'form':     return gradientForm(x, y);
    default:         return gradientRelation(x, y);
  }
}

// Gradient magnitude — decision confidence metric
export function gradientMagnitude(x, y, surfaceType = 'relation') {
  const g = gradientSurface(x, y, surfaceType);
  return Math.sqrt(g.dx * g.dx + g.dy * g.dy);
}

§9 — EXTENDING THE MANIFOLD

9.1 Adding New Substrates

To add a new substrate to any ButterflyFX application:

  1. Create [name].substrate.js following the exact class template in §5.3
  2. Register published coordinates in the constructor via this.registry.registerPublisher()
  3. Implement sample() to read needed coordinates from the registry
  4. Implement evaluate() with domain-specific manifold logic
  5. Implement publish() to write output coordinates to the registry
  6. Optionally implement render() for visual or audio output
  7. Add the substrate instance to the main loop's substrates array in main.js
  8. No changes needed to any existing substrate — the Diamond pattern guarantees zero coupling
ZERO-COUPLING GUARANTEE

Because substrates communicate only through the Registry surface, adding or removing a substrate has zero impact on any other substrate's code. The system is extensible by addition, never by modification.

§12 — GLOSSARY OF MANIFOLD TERMS

Term Definition
Manifold A mathematical space that locally resembles Euclidean space. In this architecture, the collection of surfaces upon which all computation is performed.
Surface A 2-dimensional manifold embedded in 3D space, defined by a function z = f(x, y). The primary computational primitive. All state, audio, and AI evaluation occurs on surfaces.
Substrate A self-contained computational module that owns a single domain, publishes output coordinates, and reads input coordinates from shared manifold surfaces. Substrates NEVER call each other directly.
Registry The Surface Registry — a flat, double-buffered dictionary mapping coordinate names to values. The only shared infrastructure. Contains no logic. Enforces single-writer, multi-reader access.
Coordinate A named value published to or read from the Registry. The atomic unit of inter-substrate communication. Format: domain.aspect.detail.
Gradient (∇f) The vector of partial derivatives of a surface at a point, indicating the direction of steepest ascent. Used by AI agents for decision-making and optimization.
φ (Phi) The golden ratio: (1 + √5) / 2 = 1.6180339887… The universal proportional constant governing all Manifold dimensions.
Diamond Pattern The substrate communication pattern derived from the Schwartz D-surface topology. Substrates communicate exclusively through the Registry surface — no direct coupling.
Potential State of a coordinate that exists in mathematical space but has not been evaluated.
Null State of a coordinate that has been addressed but carries no manifest value.
Manifest State of a coordinate that has been evaluated and carries a concrete value.
Dimensional Helix The ButterflyFX 7-layer architecture where each layer contains all lower layers. Layer 7 completion becomes the Layer 1 Spark of the next spiral.
Saddle Surface A surface that curves upward in one direction and downward in another. The Relation surface z = x · y is a saddle surface (hyperbolic paraboloid).
Double Buffer The Registry's frame-consistency mechanism: writes go to the back buffer; the front buffer is swapped in at the end of each frame.
